# プロジェクト3：ログファイルストレージと分析パイプライン

## 概要
このプロジェクトでは、アプリケーションやサーバーから出力されるログファイルをS3に集約し、保存・分析するシステムを構築します。開発環境でのシンプルなログ保存から始まり、本番環境での大規模なログ処理、複数ソースからの統合、自動分析へと段階的に発展します。このプロセスを通じて、大規模データの管理と処理の基礎を学びます。

## 基本バージョン：ログファイルの保存と構成

### 目的
サーバーやアプリケーションから生成されるログファイルをS3に保存し、日付やアプリケーション名に基づいてフォルダ分けして管理する。

### 使われる概念
- **S3バケットの作成**：ログファイル専用のバケットを作成
- **オブジェクトキーの設計**：`logs/application-name/YYYY/MM/DD/filename.log` などの階層的なキー構造を作成
- **AWS CloudWatch Logs の統合**：CloudWatch Logs から S3 へのエクスポート設定
- **ログファイルのアップロード**：AWS CLI または SDK を使用してログファイルを定期的にアップロード
- **ロールと権限**：EC2 インスタンスや Lambda 関数が S3 にログを書き込む権限を付与
- **タイムスタンプ**：ログファイルに生成時刻をメタデータとして記録

### 学習ポイント
- ログ設計とデータの階層化
- AWS IAM の実践的な使用法
- CloudWatch の基礎
- オブジェクトキーの設計パターン

---

## 中級バージョン：ログの圧縮、フィルタリング、複数ソースの統合

### 目的
複数のアプリケーションやサーバーからのログを1つのバケットに集約し、ストレージ効率を向上させるために圧縮を施し、特定のパターンに基づいてログをフィルタリングして、関連性の高い情報を抽出する。

### 追加される概念
- **ログの圧縮**：gzip や bzip2 などの圧縮形式を使用してファイルサイズを削減
- **フィルタリングとクエリ**：S3 Select を使用して、CSV や JSON 形式のログから特定のフィールドやレコードを効率的に抽出
- **複数ソースの統合**：異なるアプリケーション、リージョン、環境からのログを単一のバケットに統合
- **タグベースの分類**：ログファイルにタグを付けて、アプリケーション、環境（本番/開発）、ログレベルなどで分類
- **SNS/SQS との連携**：新しいログファイルがアップロードされたときに自動的にメッセージ送信
- **Athena の統合基礎**：S3 に保存されたログを SQL クエリで分析する準備
- **メタデータの検索**：オブジェクトメタデータを活用した効率的なログ検索

### 学習ポイント
- データ圧縮とストレージ効率化
- イベント駆動型アーキテクチャの実装
- 異なるデータソースの統合パターン
- データクエリの基礎（S3 Select）
- ログレベルと監視戦略

## 発展バージョン：自動分析パイプラインと BI 連携

### 目的
ログをS3に保存した後、自動的にデータ変換・分析プロセスを実行し、集計結果を別バケットに保存し、Amazon QuickSight などのBI ツールで可視化して、リアルタイムのインサイト活用を実現する。

### 追加される概念
- **Lambda による自動処理**：新しいログがアップロードされたら、Lambda 関数が自動的に実行され、ログの解析・変換を実施
- **AWS Glue**：データカタログを作成してメタデータを管理し、ログの構造を自動認識
- **Amazon Athena**：S3 上の構造化ログに対して SQL クエリを実行し、複雑な分析を実施
- **Amazon Redshift Spectrum**：大規模なログデータを Redshift で集計分析
- **ライフサイクルポリシーの高度な設定**：ログの年代に基づいて、標準ストレージから Glacier へ段階的に移行
- **パーティション設計**：年月日別などのパーティションを活用して、クエリ性能を最適化
- **結果の保存と再利用**：分析結果をパーケットなどの効率的な形式で保存
- **CloudWatch Insights との連携**：リアルタイムなログ監視と異常検知
- **コスト分析**：Athena や Redshift のクエリコストを監視し、最適化
- **アクセス制御の強化**：特定のグループのみがログを閲覧できるよう、IAM や S3 オブジェクトレベルのアクセス制御を設定

### 学習ポイント
- データパイプラインの設計と実装
- ETL（抽出、変換、ロード）プロセスの実践
- ビッグデータ処理のための分散システム
- BI ツールとのデータ連携
- 大規模なデータ分析における性能最適化
- 監視ダッシュボード構築
- コスト管理と最適化
