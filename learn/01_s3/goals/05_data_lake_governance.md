# プロジェクト5：データレイク構築とデータガバナンス

## 概要
このプロジェクトでは、複数のソースから異なるフォーマットのデータを収集し、S3に一元的に保存するデータレイク（Data Lake）を構築します。初期段階では異なるファイル形式のデータを整理して保存し、段階的にデータの品質管理、スキーマ管理、アクセス制御、カタログ化、そして自動クレンジングなどのガバナンス機能を追加していきます。このプロセスを通じて、エンタープライズレベルのデータ管理の実践を学びます。

## 基本バージョン：マルチソースデータの保存と階層化

### 目的
複数のデータソース（CSV、JSON、Parquet、XML など）からデータを収集し、S3に整理して保存する基本的なデータレイク構造を構築。

### 使われる概念
- **S3バケットの作成**：データレイク専用のバケットを作成
- **ゾーン設計**：Raw（未処理）、Processed（処理済み）、Analytics（分析用）などの複数のプレフィックスで層別化
- **ファイルフォーマット**：CSV、JSON、Parquet、ORC などの異なるフォーマットを理解し、使い分け
- **命名規則の設計**：`datasource/dataset/YYYY/MM/DD/filename.ext` のような一貫性のある命名規則を定義
- **メタデータ管理**：データソース名、取得日時、バージョン番号などを オブジェクトメタデータやタグで記録
- **アクセス制御**：異なるチーム（データエンジニア、データサイエンティスト、ビジネスユーザー）に基づく権限設定
- **イベント通知**：新しいデータがアップロードされたときに通知を送信

### 学習ポイント
- データレイクアーキテクチャの基本
- 多様なデータフォーマットの特性
- メタデータの重要性
- IAM ベースのアクセス制御
- データガバナンスの初期段階

## 中級バージョン：スキーマ管理、データ品質チェック、ETL オートメーション

### 目的
データの構造をスキーマとして定義・管理し、品質チェックルールを適用してデータの一貫性を確保し、AWS Glue などを使用してETLプロセスを自動化する。

### 追加される概念
- **AWS Glue Catalog**：データセットのメタデータを一元管理し、スキーマ情報を記録。テーブル定義を一度行えば、Athena や Redshift から直接利用可能
- **スキーマ管理**：Apache Avro や Protocol Buffers などのスキーマフォーマットを使用して、データ構造を明示的に定義・バージョン管理
- **データ品質ルール**：欠落値の許容度、データ型の検証、値の範囲チェック、重複検出などのルールを定義
- **AWS Glue Jobs**：Python や Spark を使用した ETL ジョブを実装。Raw ゾーンから Processed ゾーンへの自動変換
- **DynamicFrame**：Spark DataFrames の発展形で、スキーマが不均一なデータを柔軟に処理
- **ジョブの監視と通知**：Glue Jobs の実行状況を CloudWatch で監視し、失敗時は SNS で通知
- **キュレーション**：データの正確性と有用性を確保するため、社内チームがデータの品質を確認・承認
- **データ系統追跡**（Lineage）：どのソースデータからどの処理経由で、現在のデータが生成されたかを追跡可能に

### 学習ポイント
- スキーマの設計と管理
- データ品質フレームワーク
- AWS Glue による ETL 自動化
- Spark と分散データ処理の基礎
- メタデータ管理と検索性
- データ系統とコンプライアンス

## 発展バージョン：AI/ML統合、リアルタイム処理、高度なコンプライアンス管理

### 目的
機械学習パイプラインを構築してデータの異常検知や予測モデルを実装し、ストリーミングデータのリアルタイム取り込みを開始し、規制要件に対応するため個人情報の自動検出・マスキング、監査ログ、アクセス制御を強化する。

### 追加される概念
- **Amazon SageMaker との統合**：S3 に保存されたデータを使用して、ノートブック上で機械学習モデルを開発・訓練・デプロイ
- **データの特徴量エンジニアリング**：データレイク内のデータから機械学習に適した特徴を抽出・変換
- **ストリーミングデータ取り込み**：AWS Kinesis や Kafka を使用してリアルタイムデータを取得し、S3 に段階的に保存
- **異常検知**：機械学習モデルを使用して、データレイク内の異常なレコードを自動検出
- **機密データの自動検出**：AWS Macie を使用して、S3 内の個人情報（PII）、クレジットカード番号などを自動スキャン・検出
- **データマスキング**：検出された機密情報を自動的にマスク・暗号化
- **監査ログと追跡**：誰がいつどのデータにアクセスしたか、どのような処理を行ったかを詳細に記録
- **属性ベースのアクセス制御（ABAC）**：ユーザーや部門の属性に基づいた動的なアクセス制御
- **データ分類**：機密度レベル（Public、Internal、Confidential、Restricted）などでデータを分類
- **保持ポリシー**：規制要件に基づいたデータ保持期限の設定と自動削除
- **監視と監査**：CloudTrail、CloudWatch、Config などを使用してコンプライアンス状況を監視
- **暗号化の強化**：カスタマーマネージドキー（CMK）を使用した暗号化と鍵管理
- **VPC エンドポイント**：インターネットを通さず、プライベートネットワーク経由でS3にアクセス

### 学習ポイント
- 機械学習パイプラインの設計と実装
- ストリーミングデータアーキテクチャ
- データセキュリティと個人情報保護（GDPR対応など）
- エンタープライズレベルの監査・ガバナンス
- コンプライアンス要件の実装
- 機密データの保護と管理
- 大規模データレイク運用
