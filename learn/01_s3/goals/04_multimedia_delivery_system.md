# プロジェクト4：マルチメディアコンテンツ配信システム

## 概要
このプロジェクトでは、画像やビデオなどのマルチメディアファイルをS3に保存し、効率的に配信するシステムを構築します。小規模な画像ギャラリーから始まり、複数フォーマットでの自動変換、適応的ビットレートストリーミング、高性能なグローバル配信へと段階的に発展します。このプロセスを通じて、メディア処理とコンテンツ配信の実践を学びます。

## 基本バージョン：画像ギャラリーの構築

### 目的
複数の画像ファイルをS3にアップロードし、ウェブブラウザから閲覧可能な画像ギャラリーを作成。画像は正しいサイズで保存され、HTTPで高速にアクセスできるようにする。

### 使われる概念
- **S3バケットの作成**：マルチメディア専用のバケットを作成
- **画像ファイルの保存**：JPG、PNG、WebP などの異なる形式を保存
- **オブジェクトメタデータ**：画像の解像度、撮影日時、タグなどのメタデータを記録
- **MIME タイプ**：ブラウザが正しくファイルを解釈するために、Content-Type を正しく設定
- **アクセス制御**：特定のファイルはパブリック、その他はプライベートにするなどの柔軟なアクセス制御
- **URL構造の設計**：`images/category/filename.jpg` などの論理的なキーパスを設計
- **HTTP キャッシュヘッダ**：Cache-Control ヘッダを使用して、ブラウザやプロキシ上でのキャッシュを制御

### 学習ポイント
- 画像ファイル形式とコーデックの基本
- メタデータの活用
- HTTP キャッシュメカニズム
- ウェブセキュリティと CORS（クロスオリジンリソース共有）

## 中級バージョン：画像の自動最適化と複数フォーマット対応

### 目的
アップロードされた画像を自動的に複数の解像度やフォーマットに変換し、異なるデバイスやネットワーク速度に応じた最適なサイズを配信する。また、サムネイルも自動生成してパフォーマンスを向上させる。

### 追加される概念
- **Lambda による画像処理**：S3 にアップロードされた画像をトリガーに、Lambda 関数が自動的に実行され、複数の解像度・フォーマントに変換
- **ImageMagick / LibVips**：Lambda 内で使用できる画像処理ライブラリ
- **レスポンシブ画像の対応**：モバイル用に小さな画像、デスクトップ用に大きな画像を自動生成
- **WebP フォーマット**：より効率的な圧縮形式への自動変換
- **サムネイル生成**：ギャラリーの読み込み高速化のため、小さなサムネイル画像を自動作成
- **メタデータの抽出**：EXIF 情報から撮影日時やカメラ情報を抽出して、検索・フィルタリングに活用
- **進捗の追跡**：処理状況を DynamoDB や他のデータベースに記録
- **エラーハンドリング**：処理失敗時の通知と再試行機構

### 学習ポイント
- イベント駆動型アーキテクチャの実装
- Lambda の実行環境とリソース制限
- 画像処理の自動化とワークフロー
- メタデータの抽出と活用
- 非同期タスク処理

## 発展バージョン：CloudFront 統合、ビデオストリーミング、AI による自動タグ付け

### 目的
CloudFrontを使用してグローバルな高速配信を実現し、ビデオファイルは適応的ビットレートストリーミング（ABR）で配信、AWS Rekognition を使用して画像の内容を自動認識してタグ付けし、検索機能を強化する。

### 追加される概念
- **CloudFront ディストリビューション**：S3 を配信元として、全世界のエッジロケーションでコンテンツをキャッシュし、レイテンシーを削減
- **オリジンアクセスアイデンティティ（OAI）**：CloudFront のみが S3 にアクセスできるよう制限し、セキュリティ向上
- **キャッシュの無効化**：新しいバージョンのファイルを公開する際に、古いキャッシュを削除
- **ビデオトランスコーディング**：AWS Elemental MediaConvert を使用して、複数の解像度・ビットレート（720p、1080p など）のビデオを自動生成
- **HLS / DASH フォーマット**：ビデオストリーミング用の標準フォーマット。ネットワーク速度に応じて最適なセグメントが選択される
- **サムネイルシートの生成**：ビデオの複数フレームから サムネイル を生成
- **AWS Rekognition**：画像認識AI により、画像に含まれるオブジェクト、顔、テキストなどを自動抽出してタグ化
- **Elasticsearch / OpenSearch との連携**：メタデータやタグを使用した高速な全文検索
- **署名付きクッキー**：特定のユーザーのみがコンテンツにアクセス可能にする（有料コンテンツ配信）
- **アクセスロギング**：CloudFront からのリクエストをログして、アクセスパターンを分析
- **コンテンツの品質制御**：異なるリージョンでの再生を監視し、CDNの性能を最適化
- **費用最適化**：キャッシュヒット率や配信データ量を監視

### 学習ポイント
- グローバルな高速コンテンツ配信アーキテクチャ
- ビデオストリーミングプロトコルと適応的ビットレート
- AI/ML による自動メタデータ生成
- セキュリティと暗号化によるコンテンツ保護
- 大規模で複雑なメディア処理パイプライン
- パフォーマンス最適化とコスト削減戦略
