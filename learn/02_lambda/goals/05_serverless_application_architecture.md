# プロジェクト5：サーバーレスアプリケーションアーキテクチャ

## 概要
このプロジェクトでは、Lambda、API Gateway、DynamoDB、Cognitoなどを組み合わせて、本格的なサーバーレスアプリケーションを構築します。認証、データ管理、ファイルアップロード、リアルタイム通信など、実際のWebアプリケーションに必要な機能を段階的に実装します。

## 基本バージョン：CRUD API とデータ管理

### 目的
API GatewayとLambda、DynamoDBを使用して、基本的なCRUD（作成、読み取り、更新、削除）操作を提供するRESTful APIを構築する。

### 使われる概念
- **API Gateway REST API**：リソースとメソッドの設定
- **Lambda関数の複数パターン**：各CRUD操作用の関数
- **DynamoDB テーブル設計**：パーティションキーとソートキーの理解
- **HTTP メソッド**：GET、POST、PUT、DELETEの実装
- **リクエスト検証**：入力パラメータのバリデーション
- **レスポンス形式**：統一されたJSONレスポンス
- **エラーハンドリング**：適切なHTTPステータスコードの返却
- **CORS設定**：フロントエンドからのアクセス許可
- **環境変数**：テーブル名などの設定管理
- **IAM ロール**：Lambda関数のDynamoDBアクセス権限
- **CloudWatch Logs**：APIアクセスログの記録

### 学習ポイント
- RESTful APIの設計原則
- DynamoDBのデータモデリング基礎
- HTTPプロトコルとステータスコード
- API設計のベストプラクティス
- エラーハンドリングパターン
- セキュリティの基本（権限管理）

## 中級バージョン：認証・認可とファイルアップロード

### 目的
Amazon Cognitoを使用したユーザー認証・認可を実装し、S3への直接ファイルアップロード機能を追加して、セキュアなアプリケーションを構築します。

### 追加される概念
- **Amazon Cognito統合**：ユーザープール、IDプールの作成
- **JWT トークン検証**：API Gatewayのオーソライザー
- **ユーザー認証フロー**：サインアップ、ログイン、パスワードリセット
- **ロールベースアクセス制御（RBAC）**：ユーザーグループと権限
- **S3 Presigned URL**：クライアントからの直接アップロード
- **ファイルメタデータ管理**：DynamoDBでのファイル情報管理
- **Lambda Authorizer**：カスタム認証ロジック
- **ユーザー固有データ**：ユーザーIDによるデータ分離
- **API キー管理**：使用量プランとクォータ
- **レート制限**：APIスロットリングの設定
- **データ検証とサニタイゼーション**：XSS/SQLインジェクション対策
- **セッション管理**：リフレッシュトークンの実装
- **マルチテナンシー**：テナント分離の基礎

### 学習ポイント
- 認証と認可の実装
- JWTトークンの仕組み
- OAuth 2.0とOpenID Connectの基礎
- セキュアなファイルアップロード
- アクセス制御のベストプラクティス
- ユーザーデータの分離と保護
- APIセキュリティ対策
- セッション管理

## 発展バージョン：フルスタックサーバーレスアプリケーション

### 目的
WebSocketによるリアルタイム通信、GraphQL API、CI/CDパイプライン、監視、コスト最適化を統合した、本番運用可能なフルスタックサーバーレスアプリケーションを構築します。

### 追加される概念
- **WebSocket API**：リアルタイム双方向通信の実装
- **AppSync（GraphQL）**：柔軟なデータクエリとサブスクリプション
- **リアルタイム通知**：DynamoDB Streamsとの連携
- **検索機能**：ElasticSearch/OpenSearch統合
- **キャッシング戦略**：API Gateway、DynamoDB DAX、ElastiCache
- **CDN統合**：CloudFrontによる静的コンテンツ配信
- **SAM/CDK**：Infrastructure as Codeによるデプロイ
- **CI/CD パイプライン**：CodePipeline、CodeBuild、CodeDeploy
- **Blue/Green デプロイメント**：Lambdaエイリアスとバージョン管理
- **カナリアデプロイ**：段階的なトラフィック移行
- **分散トレーシング**：X-Rayによるエンドツーエンド追跡
- **カスタムメトリクス**：ビジネスメトリクスの監視
- **ログ集約**：CloudWatch Logs Insightsでの分析
- **セキュリティスキャン**：自動的な脆弱性検査
- **コスト最適化**：Lambda Power Tuning、リザーブドキャパシティ
- **マルチリージョン展開**：グローバルアプリケーション
- **災害復旧計画**：バックアップとリストア戦略
- **コンプライアンス**：監査ログとデータ保護
- **パフォーマンステスト**：負荷テストとボトルネック解析
- **アラート設定**：CloudWatch Alarms、SNS通知

### 学習ポイント
- フルスタックサーバーレスアーキテクチャ
- リアルタイム通信の実装
- GraphQL APIの設計と実装
- Infrastructure as Codeの実践
- CI/CDパイプラインの構築
- デプロイ戦略（Blue/Green、カナリア）
- 分散システムの監視とトレーシング
- キャッシング戦略とパフォーマンス最適化
- セキュリティのベストプラクティス
- コスト最適化手法
- スケーラビリティとレジリエンス設計
- 運用監視とトラブルシューティング
- マルチリージョン戦略
- 災害復旧とビジネス継続性
- コンプライアンスとガバナンス

## アーキテクチャ構成例

### 基本構成
```
Client → API Gateway → Lambda → DynamoDB
```

### 中級構成
```
Client → CloudFront → API Gateway → Cognito Authorizer
                                  ↓
                              Lambda → DynamoDB
                                  ↓
                              S3 (Presigned URL)
```

### 発展構成
```
Client → CloudFront → API Gateway / WebSocket / AppSync
                          ↓
                      Cognito
                          ↓
                      Lambda (+ Layers)
                      ↓  ↓  ↓
                   DynamoDB + DAX
                   S3 + ElasticSearch
                   SQS + SNS
                          ↓
                   Step Functions
                          ↓
                   CloudWatch + X-Ray
```
