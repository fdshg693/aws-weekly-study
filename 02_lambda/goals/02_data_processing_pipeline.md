# プロジェクト2：データ処理パイプライン

## 概要
このプロジェクトでは、Lambdaを使用してデータ処理パイプラインを構築します。ファイル変換、データ集計、ETL処理など、実際のビジネスシーンで必要となるデータ処理のワークフローを、段階的に実装していきます。

## 基本バージョン：シンプルなファイル変換

### 目的
S3にアップロードされた画像やファイルを自動的に変換・加工して、別のS3バケットまたは同じバケットの別フォルダに保存する。

### 使われる概念
- **S3イベント通知**：オブジェクト作成イベントをトリガーとしてLambda起動
- **ファイルの読み取りと書き込み**：boto3（Python）またはAWS SDK for JavaScriptを使用してS3操作
- **画像処理**：Pillow（Python）やSharp（Node.js）を使用した画像リサイズ・変換
- **Lambda Layers**：画像処理ライブラリを含むレイヤーの作成
- **メモリとタイムアウトの調整**：処理内容に応じたリソース設定
- **エラーハンドリング**：ファイル形式エラーや処理失敗時の対応
- **IAMポリシー**：S3バケットへの読み取り・書き込み権限の付与
- **環境変数**：出力先バケット名などの設定を環境変数で管理

### 学習ポイント
- イベント駆動型データ処理の基礎
- AWS SDKを使用したS3操作
- バイナリデータの扱い方
- メモリ効率を考慮したプログラミング
- 非同期処理とイベントループ

## 中級バージョン：並列処理とデータ変換パイプライン

### 目的
複数のファイルを並列処理し、CSV/JSONデータの変換、集計、フィルタリングを行う本格的なデータパイプラインを構築します。

### 追加される概念
- **Step Functions統合**：複数のLambda関数を連携させたワークフロー
- **並列実行制御**：Map Stateを使用した並列処理
- **エラー処理と再試行**：Step Functionsでのリトライとキャッチ
- **DynamoDBとの連携**：処理状態の記録と管理
- **SQS/SNS統合**：メッセージキューを使用した非同期処理
- **データ検証**：入力データのバリデーション処理
- **ストリーミング処理**：大きなファイルのストリーム処理
- **カスタムメトリクス**：処理件数や処理時間の記録
- **条件分岐**：ファイルタイプや内容に応じた処理の分岐
- **Lambda Destinations**：成功・失敗時の後続処理の設定

### 学習ポイント
- ステートマシンの設計
- 並列処理とスケーラビリティ
- データパイプラインのオーケストレーション
- エラーハンドリングとリトライ戦略
- 非同期処理パターン
- データ整合性の担保
- パフォーマンスの最適化

## 発展バージョン：リアルタイムストリーム処理と機械学習統合

### 目的
Kinesis Data StreamsやDynamoDB Streamsを使用したリアルタイムデータ処理と、SageMakerやRekognitionなどのAI/MLサービスとの統合により、高度なデータ分析パイプラインを実現します。

### 追加される概念
- **Kinesis Data Streams統合**：リアルタイムストリーミングデータの処理
- **DynamoDB Streams処理**：テーブル変更の即時反映
- **バッチ処理の最適化**：バッチサイズとバッチウィンドウの調整
- **Amazon Rekognition統合**：画像・動画分析の自動化
- **SageMaker統合**：機械学習モデルの推論
- **Amazon Comprehend**：自然言語処理とセンチメント分析
- **EventBridge Pipes**：高度なデータフィルタリングと変換
- **データの永続化**：処理結果のS3、DynamoDB、RDS保存
- **Lambda Power Tuning**：コスト最適化のためのメモリ設定調整
- **Provisioned Concurrency**：コールドスタートの削減
- **VPC Lambda**：プライベートネットワークでのデータ処理
- **暗号化**：データの暗号化と鍵管理（KMS）
- **Glue統合**：データカタログとETLジョブの連携
- **Athena/Redshift連携**：処理結果の分析基盤への格納
- **コスト監視とアラート**：CloudWatch AlarmsとBudgets設定

### 学習ポイント
- リアルタイムストリーム処理アーキテクチャ
- AI/MLサービスとの統合パターン
- スケーラブルなデータパイプライン設計
- データレイクアーキテクチャ
- セキュリティとコンプライアンス
- コスト最適化手法
- 運用監視とトラブルシューティング
- パフォーマンスチューニング
- ビッグデータ処理のベストプラクティス
