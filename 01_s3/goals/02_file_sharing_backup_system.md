# プロジェクト2：ファイル共有・バックアップシステム

## 概要
このプロジェクトでは、S3を使用してファイル共有とバックアップの機能を実装します。チームメンバーや顧客とファイルを安全に共有し、重要なデータをバックアップして、データ損失から守ります。基本は単純なアップロード・ダウンロードから始まり、やがて有効期限付きリンク、バージョン管理、ストレージクラスの最適化などの高度な機能へと発展します。

## 基本バージョン：シンプルなファイルアップロード・ダウンロード

### 目的
S3バケットにファイルをアップロードして保存し、アップロードしたファイルをダウンロード可能にする最小限のシステムを構築。

### 使われる概念
- **S3バケットの作成**：バックアップ用の専用バケットを作成
- **ファイルのアップロード**：AWS CLI やコンソールからファイルをS3に保存
- **ファイルのダウンロード**：S3に保存されたファイルを取得
- **オブジェクトメタデータ**：ファイルの更新日時、サイズ、ストレージクラスなどの情報
- **アクセス制御**：特定のユーザーのみがファイルにアクセスできるように権限を設定
- **IAMロールとポリシー**：アプリケーションやユーザーがS3へのアクセス権を持つようにする

### 学習ポイント
- S3の基本的なCRUD操作（作成、読み取り、更新、削除）
- AWS IAMの基礎（ユーザー、ロール、ポリシー）
- 最小権限の原則によるアクセス制御

---

## 中級バージョン：有効期限付きリンクとバージョン管理

### 目的
アップロードしたファイルへのアクセスを一時的に許可する署名付きURLを生成し、限られた時間だけファイル共有を可能にします。また、ファイルの複数バージョンを保持して、誤った上書きから回復できるようにします。

### 追加される概念
- **署名付きURL（Presigned URL）**：一定時間だけ有効なURLを生成して、公開していないファイルへの一時的なアクセスを許可
- **バージョニング**：各オブジェクトの複数バージョンを自動的に保持し、削除されたものを復元できるようにする
- **オブジェクトロック**：特定のファイルが削除や上書きされないよう保護する（コンプライアンス要件に対応）
- **タグ付け**：ファイルにメタデータタグを付けて、分類や管理を容易にする
- **S3 の イベント通知**：ファイルがアップロードされた時に Lambda や SNS を使用して自動処理を開始
- **オブジェクト の検索とフィルタリング**：メタデータやタグを基に ファイルを検索・整理

### 学習ポイント
- セキュリティトークンと署名の概念
- ファイルバージョン管理の重要性
- イベント駆動型アーキテクチャの基礎
- タグによるメタデータ管理

## 発展バージョン：自動バックアップ、ストレージ最適化、クロスリージョンレプリケーション

### 目的
ライフサイクルポリシーで古いファイルを自動的に安価なストレージクラスに移行させ、複数のAWSリージョンにデータをレプリケートして、災害対応を実現します。さらに、CloudWatch で監視し、アラートを設定して、バックアップシステムの健全性を保ちます。

### 追加される概念
- **ライフサイクルポリシー**：一定日数経過したファイルを標準ストレージから Glacier（低コストの長期保存用）に自動移行
- **ストレージクラス**：S3 Standard（標準、高速アクセス）、S3-IA（低頻度アクセス、低コスト）、Glacier（アーカイブ、最低コスト）など、ユースケースに応じた選択
- **クロスリージョンレプリケーション（CRR）**：別のリージョンの別バケットに自動的にデータをコピーし、地域的な災害に対応
- **MFA Delete**：重要なファイルの削除に多要素認証を必須にして、誤削除を防止
- **S3 Inventory**：バケット内のすべてのオブジェクトのリストをレポート化し、監査やコンプライアンス管理に利用
- **CloudWatch メトリクスと ログ**：アップロード数、ダウンロード数、ストレージ使用量などを監視
- **AWS Backup との統合**：一元的なバックアップ管理を実現
- **暗号化**：保存時（Server-Side Encryption）と転送時の暗号化を設定

### 学習ポイント
- ストレージコストの最適化戦略
- ディザスタリカバリーと事業継続計画（BCP）の概念
- データガバナンスとコンプライアンス要件
- 監視とアラート設定による運用管理
- セキュリティ多層防御
